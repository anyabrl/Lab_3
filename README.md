* **Контроль версий:** Git, GitHub (использованы Pull Requests, Code Review, разрешение конфликтов).

## Реализация принципов ООП и SOLID
Архитектура приложения спроектирована с четким разделением ответственности:
* **Инкапсуляция:** Данные строго скрыты внутри свойств моделей (`TemperatureRecord`, `CrimeRecord`), доступ к ним осуществляется только через геттеры и сеттеры.
* **Полиморфизм:** Единый класс `ForecastService` применяется для расчета скользящей средней как для температуры, так и для преступности.
* **SOLID (Принцип единой ответственности - SRP):** Визуальные компоненты (`UserControl`) отвечают только за отрисовку интерфейса. Вся бизнес-логика вынесена в отдельные классы-сервисы (`TemperatureAnalyzer`, `CrimeAnalyzer`).
* **SOLID (Принцип инверсии зависимостей - DIP):** Формы не реализуют логику внутри обработчиков кнопок, а делегируют выполнение специализированным сервисам.

## Структура веток и организация работы
Проект разработан с использованием распределенной модели Git:
* `master` — финальный стабильный релиз программы.
* `dev` — ветка интеграции, куда сливались наработки обоих участников.
* `feature/var-3-temperature` — изолированная ветка Участника 1.
* `feature/var-13-crime` — изолированная ветка Участника 2.

**Процесс работы:**
1. Каждый участник создавал отдельную ветку от `dev`.
2. Разработка велась параллельно с периодическими осмысленными коммитами (от каждого участника минимум по 5 штук).
3. Интеграция происходила строго через **Pull Requests** с описанием изменений.
4. Был проведен взаимный **Code Review** (в PR-ах оставлены комментарии с советами и замечаниями по коду).

## Вариант 3: Температура в городе
Пользователь открывает файл с данными о температуре за каждый день в течение месяца.

* **Вывод данных:** Информация выводится на экран в удобном табличном формате (`DataGridView`).
* **Построение графиков:** По данным построены графики зависимости от дня. **Минимальная температура окрашена в синий цвет, максимальная — в красный.**
* **Аналитика:** Программа автоматически вычисляет, в какие дни перепад температуры был самый сильный и самый слабый.
* **Прогнозирование:** Реализована экстраполяция методом скользящей средней на последующие **N дней** (число N указывается пользователем в специальном поле). Прогнозные данные выведены на графике **другим цветом (пунктиром)**.
* **Интерактивность и экспорт:** Графики поддерживают масштабирование колесиком мыши. Реализован **экспорт графиков в формат PNG**.

## Вариант 13: Преступность в России
Пользователь открывает файл с данными о преступности в России за последние 15 лет.

* **Вывод данных:** Информация выведена на экран в удобном табличном формате.
* **Построение графиков:** По данным построен график зависимости от года.
* **Аналитика:** Программа автоматически вычислила, какой вид преступности за 15 лет снизился больше всего, а какой меньше всего.
* **Прогнозирование:** Реализовано статистическое прогнозирование методом экстраполяции по скользящей средней на последующие **N лет**. Прогноз выведен на графике **другим цветом**.
* **Интерактивность и экспорт:** Графики интерактивные (поддерживают масштабирование). Реализован **экспорт графиков в любой формат (PNG)**.

## Инструкция по запуску
1. Клонировать репозиторий на локальный компьютер.
2. Открыть файл `Lab3.sln` в Visual Studio 2022.
3. Восстановить NuGet-пакеты (при необходимости).
4. Запустить приложение (клавиша `F5`).
5. Перейти на вкладку нужного варианта и нажать кнопку «Открыть файл».
6. Выбрать один из приложенных JSON-файлов с тестовыми данными.
7. Использовать текстовое поле и кнопку «Построить прогноз» для генерации экстраполяции.
8. Использовать кнопку «Экспорт графика» для сохранения изображения.
